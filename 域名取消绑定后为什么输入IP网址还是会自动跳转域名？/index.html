域名取消绑定后为什么输入IP网址还是会自动跳转域名

可能原因及解决方案

域名服务商未彻底删除IP绑定

如果你的域名服务商没有彻底删除IP绑定信息，IP网址可能依旧与域名关联。你可以检查并删除所有绑定到该IP地址的域名。

域名服务商缓存未清理

有时，域名服务商的缓存未及时清理，导致IP地址仍会重定向到域名。请联系你的服务商以清理相关缓存。

服务器上的 URL 重写规则未修改

服务器上的 URL 重写规则（如 Nginx 或 Apache 的 .htaccess 文件）可能未按照正确的地址进行修改，导致IP地址仍然重定向到域名。

示例：使用 Cloudflare 进行检查和调整

  1. 打开 Cloudflare 网站，进入 DNS 记录页面，检查是否还有其他域名绑定到你的服务器IP地址，确保所有无关的域名记录都已删除。

  2. 进入 缓存 - 配置 - 清除缓存，选择 清除所有内容 以确保缓存已被清理。

  3. 打开你的Nginx配置文件，确认是否已将绑定的域名更改为IP地址。这样一来，你的IP网址就能正常打开了。

以上解决方案基于80端口已开放的公网IP地址。
